⚠️ Problemi critici di sicurezza e stabilità

1. Uso errato dell’operatore bitwise | invece di logico ||

c
if (!*head | !*tail)

  • ❌ Questo è un errore: | fa un OR bit a bit, non logico.

  • ✅ Dovrebbe essere:

    c
    if (!*head || !*tail)

    Altrimenti il comportamento è imprevedibile (può accedere a memoria non valida).

2. delete_group — loop infinito e use-after-free

Nella funzione:

c
while (curr)
{
  if (strcmp(curr->name, target_name) == 0)
  {
      if (curr == *tail)
        *tail = prev;
      if (curr == *head)
        *head = curr->next;
      else
        prev->next = curr->next;
      free(curr->name);
      free(curr->garbage);
      free(curr);
  }
  prev = curr;
  curr = curr->next;
}

  • Dopo il free(curr), la riga prev = curr; curr = curr->next; usa memoria già liberata → use-after-free.

  • Inoltre, curr->next non è più accessibile dopo free(curr).

  • ✅ Soluzione:

    c
    t_group *tmp = curr->next;
...
free(curr);
curr = tmp;

  • Inoltre, se vengono liberati più elementi con lo stesso nome, il ciclo potrebbe comportarsi in modo non definito.

3. safe_new_group — buffer overflow su group->name

c
strcpy(group->name, name);

  • ❌ Non è sicuro. Non si sa quanto grande sia group->name.

  • Se t_group non alloca dinamicamente name, questa riga è una vulnerabilità di overflow.

  • ✅ Meglio allocare e copiare in sicurezza:

    c
    group->name = strdup(name);
if (!group->name) { free(group); return NULL; }

    oppure usare strncpy() se group->name è un buffer noto e limitato.

4. safe_new_group — uso di get_group() errato

c
if (get_group(head, name))

get_group accetta t_group *, ma qui passi t_group **.
Questo non compila (o produce warning) — devi dereferenziare:

c
if (get_group(*head, name))

5. Mancato controllo degli malloc()

Ogni malloc() dovrebbe essere seguito da un controllo:

c
group = malloc(sizeof(t_group));
if (!group)
    return NULL;

Altrimenti, in caso di errore di allocazione → segfault.

6. cleanup_groups_list() — cancellazione durante iterazione

Questa funzione chiama:

c
delete_group(head, tail, prev);

ma delete_group si aspetta un char *target_name, non un puntatore a t_group.
→ Tipo errato, e comportamento indefinito.

7. allocation_handler() chiamato senza contesto sicuro

Non è mostrato, ma se allocation_handler() si aspetta che group->garbage sia valida, alcune chiamate (come nel cleanup) potrebbero rompere l’invariante della lista.

8. Mancanza di sincronizzazione (race condition)

Se questa libreria venisse usata in un contesto multi-thread:

  • static t_group *groups_head, *groups_tail;
    → variabili globali condivise senza lock.
    Quindi concorrenza = corruzione di memoria assicurata.

  • ✅ Usare pthread_mutex_t per proteggere le operazioni sulle liste.

9. Error handling incompleto

Molti commenti tipo // set error → ma non viene impostato nulla.
Meglio implementare un sistema coerente di gestione errori o errno.

✅ Suggerimenti per mitigare vulnerabilità

  1. Correggere tutti gli if (!*head | !*tail) → ||.

  2. Non usare free() e poi accedere alla variabile.

  3. Usare strdup() o strncpy() invece di strcpy().

  4. Controllare sempre malloc() e restituire NULL se fallisce.

  5. Evitare globali statiche non protette.

  6. Rimuovere ambiguità di tipo tra delete_group() e cleanup_groups_list().

  7. Aggiungere logging o un enum di errori per i casi // set error.

1. cleanup_groups_list() — cancellazione durante iterazione

Nel ciclo:

c
while (curr)
{
    allocation_handler(NO_BYTES, CLEANUP, NO_DATA, curr->garbage);
    prev = curr;
    curr = curr->next;
    delete_group(head, tail, prev->name);
}

👉 Problema:
delete_group() libera la memoria di prev (e quindi prev->name), ma poi il ciclo torna a usare curr = curr->next, che potrebbe puntare in memoria già liberata.

Anche se funziona “per caso”, è comportamento indefinito.
Ricorda che curr è legato a prev attraverso i puntatori della stessa lista che stai modificando mentre la percorri.

✅ Soluzione più sicura:
Non chiamare delete_group() dentro lo stesso ciclo che scorre la lista.
Basta fare la pulizia manuale:

c
static void cleanup_groups_list(t_group **head, t_group **tail)
{
    t_group *curr = *head;
    t_group *next;

    while (curr) {
        next = curr->next;
        allocation_handler(NO_BYTES, CLEANUP, NO_DATA, curr->garbage);
        free(curr->name);
        free(curr->garbage);
        free(curr);
        curr = next;
    }

    *head = NULL;
    *tail = NULL;
}

➡️ Molto più semplice, nessun rischio di use-after-free, e fa la stessa cosa.

2. safe_new_group() — manca inizializzazione di campi

c
group->name = strdup(name);

Ma non azzeri gli altri campi (garbage, next, ecc.).

✅ Aggiungi:

c
group->garbage = NULL;
group->next = NULL;

Questo evita di passare puntatori non inizializzati a allocation_handler() o in altri punti.

3. ezg_alloc_handler() — variabili statiche globali

c
static t_group *groups_head;
static t_group *groups_tail;

❗ Se il codice può essere usato in contesto multithread → race condition.
Più thread potrebbero modificare la lista contemporaneamente.

✅ Se non puoi garantire contesto single-threaded:

  • Usa pthread_mutex_lock() / pthread_mutex_unlock() attorno alle sezioni critiche.

4. mode handling — controllo logico fragile

La gestione dei mode è un po’ confusa:

c
if (mode == CLEANUP)
    cleanup_groups_list(...);
else if (!name)
    return NULL;

if (mode == CREATE_GROUP)
    safe_new_group(...);
else if (!(group = get_group(...)))
    return NULL;

Questo significa che se mode == CLEANUP, esegue la pulizia ma poi continua nel resto della funzione.

✅ Serve un return dopo cleanup_groups_list():

c
if (mode == CLEANUP) {
    cleanup_groups_list(&groups_head, &groups_tail);
    return NULL;
}

c

c

Se name può provenire da input utente non validato, strcmp() è sicuro solo se entrambe le stringhe sono terminate da \0.
Dato che usi strdup() per assegnare name, va bene, ma ricordalo se la funzione verrà riutilizzata altrove.

c
#include "ezalloc.h"
#include "ezalloc_internal.h"

/* ------------------------- Helpers per gestione gruppi ------------------------- */

/* Pulizia completa della lista */
static void cleanup_groups_list(t_group **head, t_group **tail)
{
    t_group *curr = *head;
    t_group *next;

    while (curr)
    {
        next = curr->next;
        allocation_handler(NO_BYTES, CLEANUP, NO_DATA, curr->garbage);
        free(curr->name);
        free(curr->garbage);
        free(curr);
        curr = next;
    }

    *head = NULL;
    *tail = NULL;
}

/* Rimuove un gruppo specifico dalla lista */
static void delete_group(t_group **head, t_group **tail, const char *target_name)
{
    t_group *curr = *head;
    t_group *prev = NULL;

    while (curr)
    {
        if (strcmp(curr->name, target_name) == 0)
        {
            if (curr == *head)
                *head = curr->next;
            else
                prev->next = curr->next;

            if (curr == *tail)
                *tail = prev;

            free(curr->name);
            free(curr->garbage);
            free(curr);
            return;
        }
        prev = curr;
        curr = curr->next;
    }
}

/* Restituisce il puntatore al gruppo con nome specificato */
static t_group *get_group(t_group *head, const char *name)
{
    while (head)
    {
        if (strcmp(head->name, name) == 0)
            return head;
        head = head->next;
    }
    return NULL;
}

/* Crea un nuovo gruppo se non esiste già */
static t_group *create_group_safe(t_group **head, t_group **tail, const char *name)
{
    if (get_group(*head, name))
        return NULL;

    t_group *group = malloc(sizeof(t_group));
    if (!group)
        return NULL;

    group->name = strdup(name);
    group->garbage = NULL;
    group->next = NULL;

    if (!*head)
    {
        *head = group;
        *tail = group;
    }
    else
    {
        (*tail)->next = group;
        *tail = group;
    }

    return group;
}

/* ------------------------- Funzione pubblica ------------------------- */

void *ezg_alloc_handler(size_t size, int mode, void *target, char *name)
{
    static t_group *groups_head = NULL;
    static t_group *groups_tail = NULL;
    t_group *group;

    switch (mode)
    {
        case CLEANUP:
            cleanup_groups_list(&groups_head, &groups_tail);
            return NULL;

        case CREATE_GROUP:
            create_group_safe(&groups_head, &groups_tail, name);
            return NULL;

        case DELETE_GROUP:
            group = get_group(groups_head, name);
            if (group)
            {
                allocation_handler(size, CLEANUP, NO_DATA, group);
                delete_group(&groups_head, &groups_tail, name);
            }
            return NULL;

        case RELEASE_GROUP:
            group = get_group(groups_head, name);
            if (group)
                allocation_handler(size, CLEANUP, NO_DATA, group);
            return NULL;

        default: /* ALLOC / FREE / altre operazioni */
            group = get_group(groups_head, name);
            if (!group)
                return NULL; // group non trovato
            return allocation_handler(size, mode, target, group);
    }
}
